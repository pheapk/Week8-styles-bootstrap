How HTML, CSS, and JavaScript work together

Frontend code consists of three main parts.

  • HTML
  • CSS
  • JavaScript.

If you compare parts to a human body, HTML would be the skeleton that holds the body up, CSS would be the body's outward appearance, and JavaScript would be the muscular system that lets the body move.

HTML provides the basic components of a webpage, with individual pieces called elements, declared using tags or classes. Here are some examples of HTML elements used for simple layout. 


<h1>I'm a large header element.</h1>
<h2>I'm a slightly smaller header element.</h2>

<p>I'm a paragraph element. You can write lots of text here. </p>

<p class="specific-paragraph">I'm another paragraph.  I have a a specific class name.
</p>

<button>Click Me!</button>

<div class="container>
  <ul class="grocery-list">
    <li>Milk</li>
    <li>Eggs</li>
    <li>Coffee</li>
  </ul>
</div>

CSS styles the HTML elements to the specifications you give it. You can target HTML elements in CSS via selectors, which are pre-declared HTML tags, class names, IDs, or any combination thereof. A grouping of a selector and CSS properties applied on that selector is called a rule declaration. Here are some examples. 

h1 {
  background: orange;
  color: blue;
}

h2 {
  background: green;
  border: 1px solid red;
}

p {
  font-family: courier,arial,helvetica;
}

.specific-paragraph {
  font-family: "Times New Roman", Times, serif;
  font-weight: 5em;
}

.container { 
  background: pink;
  text-color: white;
}

.container,
.grocery-list {
   list-style-type: square;
}

While there are frameworks built on top of these parts that allow you to interact with them faster, like Bootstrap, SASS, and React, the basic functionality of each of the parts remains the same.

Keeping files separate

Style sheets and JavaScript code eventually become too large to manage inside of your HTML document and, as such, need to go into their own separate documents. You link those documents in different parts of your HTML.

CSS style sheets link via the <head> section of your HTML file. 

<html>
    
    <head>
      <title>My Webpage</title>
      <link rel=”stylesheet” href=”styles.css”>
    </head>
    
    <body> ... </body>
    
    </html>

JavaScript files link at the end of your HTML document, right before the closing </body> tag.

<html>
    
    <head>
      <title>My Webpage</title>
    </head>
    
    <body> 
      <!--Code here-->
      <script src =”script.js”></script>
    </body>
    
    </html>

To check if your files have been linked properly, you can:

  • Change a color property in your CSS file, and,
  • Change a console command in your JavaScript file.

If the files are linked properly, you should see the changes reflected when you re-load your webpage.

Properties

The individual elements we use to style CSS are called properties. There are hundreds of properties, such as color, font-size, border, background-color, and so forth, so don't expect to memorize them all. You can see a list of common CSS properties here.

Here is an example of a container, styled using several properties, such as color, border, padding, and others. 

.container { 
          display: grid;:
          text-align: center;
          font-size: 7em;
          font-family:'Lucida Sans', 'Lucida Sans Regular', 'Lucida Grande', 'Lucida Sans Unicode', Geneva, Verdana, sans-serif;
          -webkit-text-stroke: 5px black;
          color: white;
          padding: 5px;
          background-image: linear-gradient(to bottom right, red, yellow);
          border: 10px solid black;
    }

Grid

Grid has risen to prominence as a way to overcome some of the limitations of basic CSS by providing an intelligent way to lay out your HTML elements on the page quickly and with fewer headaches.

To style elements with grid, it is best to wrap the elements you want to style in a wrapper, named something like "container" or "content" as a class. You can then style the elements using grid. Here is an example.

First, you can identify the HTML elements that you want to style: 

<main class="content">
      <div>1</div>
      <div>2</div>
      <div>3</div>
      <div>4</div>
      <div>5</div>
      <div>6</div>
      <div>7</div>
      <div>8</div>
      <div>9</div>
      <div>10</div>
      <div>11</div>
      <div>12</div>
    </main>

Then, write some basic styling on the elements themselves, using color, etc:

      div {
          background: cadetblue;
          padding: 30px;
          text-align: center;
          font-size: 40px;
          color: darkblue; 
        }
        div:nth-of-type(odd) {
          background: burlywood;
        }

Then, you can quickly and easily lay them out in a custom grid, using CSS grid on the .content class. 

   .content {
          display: grid;
          grid-template-columns: 1fr 2fr 3fr 4fr;
          grid-template-rows: 3fr 2fr 1fr;
          gap: 5px;
          border: 1px solid darkblue;
        }

Bootstrap

Bootstrap is an HTML, CSS and JavaScript framework that is incredibly useful for styling your HTML with out-of-the-box UI elements and JavaScript-powered widgets.

It's great to use it when you are on a tight deadline. It is also great to use when you want to use some tried-and-true styles. However, Bootstrap is not good to use to learn CSS, as it can be too complex and overwhelming.

To learn more about Bootstrap and get to started using some external Bootstrap files, check out the website at https://getbootstrap.com/

Webfonts

Up until now, you've been explicitly declaring a font-family to use in your CSS files like this:

p {
  font-family: Helvetica, "Trebuchet MS", Verdana, sans-serif;
}

Webfonts allow you to externally reference whole libraries of fonts that download at the same time as your CSS file loads onto the browser. This gives you an incredible number of options. However, not all fonts are cross browser compatible, which means you may have to include different versions for different browsers.

The syntax for using webfonts will look something like this at the top of your CSS file: 

@font-face {
  font-family: "myFont";
  src: url("myFont.woff");
}

And then, later in your CSS file, you can use the downloaded font anywhere you want. 

html {
  font-family: "myFont", "Bitstream Vera Serif", serif;
}

A great open library to use with webfonts is Google's fonts library, at https://fonts.google.com/.

Here are a couple of examples of webfonts being used on the same block of text. Notice the different ways you can import the fonts in the head section of the HTML document.

Goldman font family

<!DOCTYPE html>
    <html>
    
    <head>
      <title>Fonts</title>
      <style>
        @import url('https://fonts.googleapis.com/css2?family=Goldman&display=swap');
        p {
          font-family: 'Goldman', cursive;
        }
      </style>
    </head>
    
    <body>
      <p>Will you be having coffee before, during, or after you finish coding?</p>
    </body>
    
    </html>

Merriweather font family

<!DOCTYPE html>
    <html>
    
    <head>
      <title>Fonts</title>
        <link rel="preconnect" href="https://fonts.gstatic.com">
        <link href="https://fonts.googleapis.com/css2?family=Merriweather:wght@300&display=swap" rel="stylesheet">
      <style>
        p {
          font-family: 'Merriweather', serif;
        }
      </style>
    </head>
    
    <body>
      <p>Will you be having coffee before, during, or after you finish coding?</p>
    </body>
    
    </html>
